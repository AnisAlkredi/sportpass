Defensive Mobile Security & Privacy Assessment Report

Version: 1.0.3 (versionCode 3) | Assessment Date: February 22, 2026

Assessment Type: Black-box + Gray-box static analysis (dynamic checks attempted but not feasible in current environment)

Methodology Baseline: OWASP MASVS v2, OWASP MASTG, OWASP Mobile Top 10, NIST SP 800-163 Rev.1 mindset

A) Executive Summary

Overall Risk: CRITICAL

Rationale:

  • Release artifact is signed with Android Debug certificate (high supply-chain and release governance risk).
  • In-app OTA install path + package install permissions materially increase abuse impact if update controls fail.
  • Internal/private HTTP endpoints are embedded in release artifact.
  • Backup/data extraction hardening attributes are not explicitly configured.
  • Permission profile exceeds strict least-privilege for a sensitive public-service app.

Top 5 Business Findings

  1. Debug-signed release build can fail store/compliance controls and weakens trust in production distribution.
  2. Update/install channel can become a high-impact propagation path during backend or transport compromise events.
  3. Embedded internal endpoints disclose infrastructure details and increase misconfiguration/transport risk.
  4. Lack of explicit backup governance can expose sensitive user data through backup and device extraction paths.
  5. Broad permissions expand privacy and regulatory exposure footprint.

Release Recommendation

NO-GO for production release.

Conditions to proceed: production signing, OTA hardening, endpoint cleanup, backup/privacy hardening, and targeted dynamic verification on emulator/device.

D) Privacy Review

Likely Data Handled

  • Personal identity/contact details, appointment records, complaint details, OTP and auth metadata.
  • Potentially sensitive support content and device identifiers linked to push messaging tokens.

Data at Rest

  • Encrypted shared preference constructs are present.
  • Risk remains due to non-explicit backup/data extraction policy configuration.

Data in Transit

  • Production HTTPS endpoint references are present.
  • Internal HTTP endpoint references are also embedded and should be removed from production artifacts.

Trackers / SDK Sharing

  • Firebase and Google Play Services components are present (messaging/auth/location related).
  • No ad-tech SDK concentration was evident in this static pass.

Privacy Recommendations

  • Apply strict least-privilege permissions and scope runtime consent by feature need.
  • Set explicit backup/data extraction controls and retention limitations.
  • Adopt release-time privacy linting for permissions, endpoints, and sensitive logs.
